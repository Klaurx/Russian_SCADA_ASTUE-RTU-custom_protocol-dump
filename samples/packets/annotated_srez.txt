# Annotated RTOS Srez Frame Examples
#
# These are synthetic example frames constructed from the confirmed wire format.
# No real captured traffic is present in this repository. All examples are for
# documentation and parser testing purposes only.
#
# Frame structure confirmed from disassembly of MakeSrezBuf(),
# MakeSrezWithFictiveTime(), and MakeAdrBuf() in libkanaldrv.so.1.
#
# Header layout (10 bytes):
#   byte 0:     magic byte 0x75 (ASCII u, from USO)
#   bytes 1-6:  BCD timestamp YY MM DD HH MM SS
#   byte 7:     frame type discriminator
#   bytes 8-9:  reserved, always 0x0000
#
# All examples use the timestamp 2024-10-09 14:30:00, which encodes as:
#   BCD year   0x24
#   BCD month  0x10
#   BCD day    0x09
#   BCD hour   0x14
#   BCD minute 0x30
#   BCD second 0x00


# Example 1: Discrete sub-frame (type discriminator 0x01)
#
# Item stride for discrete type: 0x1c bytes (28 bytes) per item.
# Status flag at item offset +0x1b, bit 1 = data is fresh.
#
# Header:
#   00: 75   magic byte
#   01: 24   BCD year 2024
#   02: 10   BCD month October
#   03: 09   BCD day 9
#   04: 14   BCD hour 14
#   05: 30   BCD minute 30
#   06: 00   BCD second 00
#   07: 01   type: discrete sub-frame
#   08: 00   reserved low byte
#   09: 00   reserved high byte
#
# Payload: one item of 28 bytes (0x1c stride)
#   bytes 0x0a through 0x19: value data (encoding internal to tag struct)
#   byte  0x25: status flags, bit 1 set (0x02) means fresh data
#   bytes 0x26 through 0x27: padding to stride boundary
#
# Raw hex (38 bytes total: 10 header + 1 item of 28 bytes):
75 24 10 09 14 30 00 01 00 00
DE AD BE EF DE AD BE EF DE AD BE EF DE AD BE EF
DE AD BE EF DE AD 02 00 00 00


# Example 2: Impulse and counter sub-frame (type discriminator 0x09)
#
# Item stride for impulse type: 0x10 bytes (16 bytes) per item.
# Status flag at item offset +0x0c, bit 0 = data is fresh.
#
# Header:
#   00: 75   magic
#   01-06:   timestamp as above
#   07: 09   type: impulse/counter sub-frame
#   08-09:   00 00 reserved
#
# Payload: one item of 16 bytes (0x10 stride)
#   bytes 0x0a through 0x1a: counter data
#   byte  0x16: status flags, bit 0 set (0x01) means fresh data
#
# Raw hex (26 bytes total: 10 header + 1 item of 16 bytes):
75 24 10 09 14 30 00 09 00 00
00 01 F4 00 00 00 01 00 00 00 00 00 00 00 00 01


# Example 3: Analog sub-frame (type discriminator 0x02)
#
# Item stride for analog type: 0x68 bytes (104 bytes) per item.
# Status flag at item offset +0x61, bit 2 = data is fresh.
#
# Header only shown here due to the large payload size:
75 24 10 09 14 30 00 02 00 00
# followed by one or more 104-byte analog payload items


# Example 4: Normal data address frame (type discriminator 0x7f)
#
# Generated by MakeAdrBuf() when param equals 0.
# Contains only the 10-byte header with no payload items.
# Used during session establishment to exchange addressing information.
#
75 24 10 09 14 30 00 7F 00 00


# Example 5: Fictive timestamp frame (type discriminator 0x82)
#
# Generated by MakeSrezWithFictiveTime() when real GPS time is unavailable.
# The type byte 0x82 is produced in the original binary by the SBB
# (subtract with borrow) instruction sequence: 0x75 minus 0x73 with the
# carry flag set yields 0x82. This is a compiler artifact, not a manually
# chosen constant.
#
# The timestamp fields are populated from the last known TIME_SERVER_KANAL
# value rather than from a live GPS fix.
#
75 24 10 09 14 30 00 82 00 00


# Notes on Type Byte Encoding
#
# The AnalizBufPriemAndSendOtvet jump table at offset 0x4b22 in libkanaldrv.so.1
# handles command codes 0x0 through 0xd. The type discriminator in srez frames
# is separate from this command dispatch table. Frame type bytes seen in srez
# frames (0x01, 0x02, 0x09, 0x7f, 0x82) identify the I/O data type carried
# in the payload, while the command codes (0x0 through 0xd) in response frames
# identify the requested operation.
#
# The SOST_SEND struct field at byte offset +0x2 within that struct holds the
# item count for the payload. MakeSrezBuf writes this count into the buffer
# alongside the 10-byte header before appending the payload items.
